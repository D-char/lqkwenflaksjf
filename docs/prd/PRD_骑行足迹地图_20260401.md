## 2. 产品概述

### 2.1 产品愿景

> **"我的骑行足迹，点亮走过的每一公里"** —— 通过顽鹿运动API获取骑行记录，解析FIT文件提取GPS轨迹，在地图上叠加显示所有骑行轨迹，形成个人路网覆盖地图，统计去重里程和个人覆盖进度。

### 2.2 产品定位

一款面向骑行爱好者的**个人骑行足迹地图小程序**，通过对接顽鹿运动API获取骑行数据，解析FIT文件提取GPS轨迹，在地图上叠加显示所有骑行轨迹，统计去重里程和个人路网覆盖进度。仅展示用户个人数据，不提供排名功能。

**核心功能**：
- 顽鹿运动API对接（获取近三个月骑行记录）
- FIT文件解析（提取GPS轨迹坐标）
- 轨迹叠加显示（所有历史轨迹叠加形成路网）
- 全国路网地图展示（OpenStreetMap）
- 本省/本市路网总里程显示
- 用户骑行里程统计（GPS坐标点去重）
- 分享足迹地图成就

### 2.3 核心价值

| 用户需求 | 产品解决方案 |
|----------|-------------|
| 看到所有骑过的地方 | FIT文件解析 → 轨迹叠加显示在地图上 |
| 统计真实骑行里程 | GPS坐标点去重计算，不重复计数 |
| 了解本地区路网规模 | 显示本省/本市路网总里程 |
| 分享骑行成就 | 足迹地图卡片一键分享 |
| 无需手动记录 | 顽鹿运动API自动获取骑行数据 |

### 2.4 产品目标

**业务目标**：
- 6个月内达到 50,000 注册用户
- 用户平均骑行里程 ≥ 100km/月

**产品目标**：
- MVP上线周期 ≤ 2个月
- 用户留存率（30天）≥ 40%

#### F002 - FIT文件解析

| 字段 | 内容 |
|------|------|
| **功能名称** | FIT文件解析 |
| **功能描述** | 解析FIT二进制文件（Garmin FIT Protocol），提取GPS轨迹坐标数据（经纬度、时间、速度、海拔），转换为JSON格式存储。 |
| **优先级** | P0 |
| **前置条件** | FIT文件已下载到本地 |

**FIT文件格式说明**：

FIT（Flexible and Interoperable Data Transfer）是Garmin定义的二进制协议，用于存储运动数据。包含以下数据类型：
- **GPS轨迹数据**：经纬度坐标、时间戳
- **运动数据**：速度、距离、心率、功率、海拔
- **设备信息**：设备型号、固件版本

**FIT文件数据结构**（示例文件：`MAGENE_C706_2026-03-15_135032_1010432.fit`）：

FIT文件包含以下核心消息类型：
- **File Header**：文件头信息（协议版本、文件大小）
- **Session Message**：会话信息（总时长、总里程、开始时间）
- **Record Message**：GPS轨迹记录（每秒或每几秒采样一次）
  - `timestamp`：时间戳
  - `position_lat`：纬度（semicircles单位，需转换）
  - `position_long`：经度（semicircles单位，需转换）
  - `speed`：速度（m/s）
  - `altitude`：海拔（m）
  - `heart_rate`：心率（bpm）
  - `power`：功率（watts）

**坐标转换公式**：
```javascript
// FIT文件中的经纬度是semicircles单位，需转换为度
function semicirclesToDegrees(semicircles) {
  return semicircles * (180 / Math.pow(2, 31));
}

// 示例
position_lat = 894374720  // semicircles
lat = 894374720 * (180 / 2147483648) = 75.0  // 度

position_long = -1234567890  // semicircles
lon = -1234567890 * (180 / 2147483648) = -103.5  // 度
```

**解析流程**：
1. 读取FIT二进制文件
2. 解析File Header，获取协议版本
3. 遍历所有Record Message
4. 提取GPS坐标（position_lat, position_long），转换为度单位
5. 提取时间戳（timestamp），速度（speed），海拔（altitude）
6. 转换为JSON格式存储

**解析后的JSON数据**：
```json
{
  "track_id": "track_001",
  "fit_file_name": "MAGENE_C706_2026-03-15_135032_1010432.fit",
  "start_time": "2026-03-15T13:50:32Z",
  "end_time": "2026-03-15T15:50:32Z",
  "total_distance_km": 25.5,
  "device": "MAGENE C706",
  "points": [
    {
      "lat": 39.92000,
      "lon": 116.10000,
      "time": "2026-03-15T13:50:32Z",
      "speed": 15.2,
      "altitude": 50.3,
      "heart_rate": 120,
      "power": 180
    },
    {
      "lat": 39.92500,
      "lon": 116.10500,
      "time": "2026-03-15T13:50:35Z",
      "speed": 16.5,
      "altitude": 51.2,
      "heart_rate": 125,
      "power": 185
    }
  ]
}
```

**技术选型**：
- JavaScript：`fit-file-parser` 库（https://github.com/perevales/fit-file-parser）
- Python：`fitparse` 库（https://github.com/dtcooperman/fitparse）
- Go：`github.com/muktihari/fit` 库

**解析算法示例（JavaScript）**：
```javascript
import FitParser from 'fit-file-parser';

const fitParser = new FitParser({
  force: true
});

function parseFitFile(fitFilePath) {
  // 读取FIT文件二进制数据
  const fitData = fs.readFileSync(fitFilePath);
  
  // 解析FIT文件
  const result = fitParser.parse(fitData);
  
  // 提取GPS轨迹数据
  const records = result.records;
  const points = records.map(record => ({
    lat: record.position_lat * (180 / Math.pow(2, 31)),
    lon: record.position_long * (180 / Math.pow(2, 31)),
    time: new Date(record.timestamp * 1000).toISOString(),
    speed: record.speed,
    altitude: record.altitude,
    heart_rate: record.heart_rate,
    power: record.power
  }));
  
  return {
    track_id: result.activity?.activity_id,
    start_time: result.sessions?.[0]?.start_time,
    total_distance_km: result.sessions?.[0]?.total_distance / 1000,
    points: points
  };
}
```

**异常流程**：
- FIT文件损坏 → 显示"文件解析失败，跳过该记录"
- FIT文件格式不兼容 → 显示"不支持该FIT文件版本"
- GPS数据缺失 → 显示"该记录无GPS数据，跳过"

**业务规则**：
- 解析后数据存储到IndexedDB
- 每个FIT文件解析生成一条轨迹记录
- GPS坐标精度：小数点后5位
- 时间戳转换为ISO 8601格式

#### F006 - 用户骑行里程统计（GPS坐标点去重）

| 字段 | 内容 |
|------|------|
| **功能名称** | 用户骑行里程统计（去重） |
| **功能描述** | 统计用户在本省/本市内的骑行里程，采用GPS坐标点去重规则：相同的GPS坐标点只计算一次距离。 |
| **优先级** | P0 |

**去重算法**：
```python
# 伪代码
def calculate_unique_distance(user_tracks, region):
    # 1. 提取所有轨迹的GPS坐标点
    all_points = []
    for track in user_tracks:
        for point in track.points:
            if point in region:
                all_points.append((point.lat, point.lon))
    
    # 2. 坐标点去重（精度：小数点后5位）
    unique_points = set()
    for lat, lon in all_points:
        normalized = (round(lat, 5), round(lon, 5))
        unique_points.add(normalized)
    
    # 3. 计算去重后的里程
    sorted_points = sort_by_time(unique_points)
    total_distance = 0
    for i in range(len(sorted_points) - 1):
        distance = haversine_distance(sorted_points[i], sorted_points[i+1])
        total_distance += distance
    
    return total_distance
```

**去重规则**：
- GPS坐标精度：小数点后5位（约1米精度）
- 相同坐标点：`(lat, lon)` 归一化后相同视为重复
- 区域判断：根据GPS坐标判断是否在本省/本市范围内

| **优先级** | P0 |

```

路网总里程：20,000km
覆盖率 = 150 / 20000 = 0.75%
```

## 6. FIT文件解析详细说明

### 6.1 FIT文件格式规范

**FIT协议版本**：根据Garmin FIT SDK定义

**文件结构**：
```
FIT File Structure:
├── File Header (14 bytes)
│   ├── Protocol Version (1 byte)
│   ├── Profile Version (2 bytes)
│   └── Data Size (4 bytes)
│
├── Data Records
│   ├── Definition Messages (定义消息类型)
│   └── Data Messages (数据消息)
│       ├── File ID Message
│       ├── Session Message (会话信息)
│       ├── Activity Message (活动信息)
│       ├── Record Messages (GPS轨迹记录)
│       ├── Event Messages (事件记录)
│       └── Device Info Messages (设备信息)
│
└── File CRC (2 bytes)
```

### 6.2 GPS轨迹数据提取

**核心数据字段**：
- `timestamp`：时间戳（秒，Unix timestamp）
- `position_lat`：纬度（semicircles单位）
- `position_long`：经度（semicircles单位）
- `speed`：速度（m/s）
- `altitude`：海拔（m）
- `heart_rate`：心率（bpm）
- `power`：功率（watts）

**坐标转换公式**：
```javascript
// semicircles to degrees
lat_deg = position_lat * (180 / 2^31)
lon_deg = position_long * (180 / 2^31)

// 示例
position_lat = 894374720 semicircles
lat_deg = 894374720 * (180 / 2147483648) = 75.0 度
```

### 6.3 FIT文件解析示例代码

**JavaScript示例**：
```javascript
import FitParser from 'fit-file-parser';

const fitParser = new FitParser();

async function parseFitFile(fitFileBuffer) {
  try {
    const fitData = fitParser.parse(fitFileBuffer);
    
    // 提取会话信息
    const session = fitData.sessions?.[0];
    
    // 提取GPS轨迹记录
    const records = fitData.records || [];
    
    // 转换坐标
    const points = records.map(record => ({
      lat: (record.position_lat || 0) * (180 / Math.pow(2, 31)),
      lon: (record.position_long || 0) * (180 / Math.pow(2, 31)),
      time: new Date((record.timestamp || 0) * 1000).toISOString(),
      speed: record.speed || 0,
      altitude: record.altitude || 0,
      heart_rate: record.heart_rate || 0,
      power: record.power || 0
    }));
    
    return {
      track_id: fitData.activity?.activity_id || generateTrackId(),
      device: fitData.device_info?.[0]?.device_name || 'Unknown',
      start_time: session?.start_time || points[0]?.time,
      end_time: session?.timestamp || points[points.length - 1]?.time,
      total_distance_km: (session?.total_distance || 0) / 1000,
      points: points.filter(p => p.lat !== 0 && p.lon !== 0)
    };
  } catch (error) {
    console.error('FIT file parse error:', error);
    throw new Error('FIT文件解析失败');
  }
}
```
